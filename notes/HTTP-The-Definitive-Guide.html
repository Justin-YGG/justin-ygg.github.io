URL与资源

URI (Uniform Resource Identifier) 包含 URL (Uniform Resource Locator) 和 URN (Uniform Resource Name)。 目前普遍使用的是 URL，URL 指代的是资源的实际地址，如果资源被移走了，就不能再访问。URN 通过提供一个统一的资源名称来保证 无论资源被移到何处都能访问得到。PURL (Persistent Uniform Resource Locators) 就是 URN 的一个实际应用，其通过提供 一个 resource locator (中间资源定位符) 对实际的 URL 进行登记和追踪，用户可以访问一个固定的永久 URL，定位符会重定向到 资源的实际地址。

URL 语法格式: <shceme>://<user>:<password>@<host>:<port>/<path>;<param>?<query>#<frag>

  • 这里提一下 param 和 query 的区别，两者都是以键值对的形式存在即 key=value, param 出现在 path 中以 ; 隔开， query 在 path 之后，并且以 ? 与 path 隔开
  • 在 POST 请求中 param 可以在 URL 中被传到服务端， query 只能在 GET 请求的 URL 中
